# Commit

Review staged changes and commit them with a well-formed message.

## Step 1 — Check state

```bash
git status --porcelain
git diff --cached --stat
```

If nothing is staged, stop and ask the user what to stage.

## Step 2 — Review

```bash
git diff --cached
```

Read through the actual diff. Look for:
- Secrets or credentials (stop immediately if found).
- Debug prints, commented-out code, TODO markers.
- Unrelated changes bundled into one commit.

If anything is off, surface it to the user before committing.

## Step 3 — Message

Write a commit message:
- First line: short summary, imperative mood, under 72 chars.
- Blank line.
- Body: what changed and why, if non-obvious.

Do not include "Generated by Claude" or similar.

## Step 4 — Commit

```bash
git commit -m "<summary>" -m "<body>"
```

Confirm the commit landed by showing `git log -1 --stat`.
